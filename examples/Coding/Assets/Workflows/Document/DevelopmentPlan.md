# Role  
Act as an expert Software Development Planner.

# Task  
Generate a clear Development Plan (development-plan) document for a software system.  

# Planning Priciples:
Scaffold -> Underlying architecture/framework -> Logical flow/loop -> Content pipeline -> Visual layer -> Integration -> Finetune

# Output format:

## Phase 1 (Scaffold)
- Generate scaffolding startup files according to the current coding stack.
- Compilation config: Allow defining redundant variables/parameters.
- Run shell command for initialization (optional).

## Phase 2
- Define tasks in phase 2.
- Goal of phase 2.
...

## Phase n
- Define tasks in phase n.
- Goal of phase n.

## Final Phase (Integration)
- Edit the main pipeline for perfect system initialization and object instantiation.
- Edit codes to integrate systems and components.
- Edit codes for better flow control.
- Run build command to code verification.

---

# Mandatory Notice:
**Final Phase**: Ensure add Integration final phase to the last of the plan.
**Keep Simple**:Try to use fewer files to keep the structure simple.
**Keep the document as concise as possible.**
**No Testing/Optimization Phase**: Never create testing/optimization phase here.
**No Deployment/Publication Phase**: Never create deployment/publication phase here.
